## 💻 系统要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Windows 10/11 64位 |
| **手机系统** | Android 5.0+（音频转发需 Android 11+）|
| **USB 调试** | 需要开启（设置 → 开发者选项 → USB 调试）|
| **权限** | 无需 root，无需在手机安装任何应用 |

### 开启开发者选项

1. 手机 → **设置** → **关于手机**
2. 连续点击 **版本号** 7 次 → 提示「已开启开发者模式」
3. 返回设置 → **系统** → **开发者选项**
4. 打开 **USB 调试**

> 部分手机（如小米）还需在开发者选项中开启 **「USB 调试（安全设置）」** 才能使用键鼠控制。

## 🚀 快速上手

### 安装

1. 从 [官网](https://hyxjx2006.github.io/scrcpy-gui/) 或 [GitHub Release](https://github.com/HYXJX2006/scrcpy-gui/releases/latest) 下载安装包
2. 双击安装，选择安装目录
3. 安装完成后双击桌面快捷方式启动

### 首次使用

```mermaid
graph LR
    A[启动应用] --> B[USB 连接手机]
    B --> C[设备自动识别]
    C --> D[点击设备选中]
    D --> E[点「投屏」→开始投屏]
```

## 🔌 USB 有线连接

**最简单、延迟最低的方式。**

1. 用数据线将手机连接至电脑
2. 手机上允许 **USB 调试**
3. 打开 scrcpy GUI → 左侧自动显示设备
4. 点击设备 → 侧边栏「投屏」→ **开始投屏**

> 首次连接时，手机会弹出「允许 USB 调试吗？」对话框，勾选「始终允许」后确定。

## 📶 WiFi 无线连接

### 方法一：ADB 无线配对（Android 11+ 推荐）

1. 手机通过 USB 连接一次电脑（后续无需 USB）
2. 运行以下命令让手机监听 ADB 端口：
   ```
   adb tcpip 5555
   ```
3. 拔掉 USB 线
4. 在 scrcpy GUI 首页输入手机 IP 地址
5. 点「连接」→ 去「投屏」页点「开始投屏」

### 方法二：无线调试配对（无需 USB）

1. 手机 → **开发者选项** → **无线调试** → 开启
2. 点 **「使用配对码配对设备」** → 记录 IP 端口和配对码
3. 电脑运行：
   ```
   adb pair <IP>:<端口>
   ```
4. 输入配对码 → 配对成功
5. 在 scrcpy GUI 输入无线调试页面显示的**服务端口**（非配对端口）
6. 点击「连接」→ 开始投屏

### 获取手机 IP 地址

- **方法一**：手机 → 设置 → 关于手机 → 状态信息 → IP 地址
- **方法二**：查看路由器管理页面
- **方法三**：USB 连接后运行 `adb shell ip addr show wlan0`

> 💡 无线连接延迟比 USB 高，适合日常使用，游戏建议用 USB。

## 🎮 投屏控制

连接成功后，控制栏提供以下功能：

| 按钮 | 功能 | 说明 |
|------|------|------|
| 🔲 **全屏** | 设置下次投屏全屏 | 当前版本暂不支持运行时切换 |
| 💤 **息屏/亮屏** | 切换手机屏幕 | 通过 ADB 模拟电源键 |
| 🔄 **旋转** | 横屏/竖屏切换 | 通过 ADB 设置系统旋转 |
| 📷 **截屏** | 保存手机屏幕截图 | 保存到 `图片/scrcpy-screenshots/` |
| ⏹ **停止** | 退出投屏 | 关闭 scrcpy 窗口 |

## ⚙️ 设置说明

### 视频设置

| 选项 | 说明 | 推荐值 |
|------|------|--------|
| **分辨率限制** | 限制画面尺寸，越小越流畅 | 游戏 1024，演示 原始尺寸 |
| **帧率限制** | 限制每秒帧数 | 日常 30fps，游戏 60fps |
| **视频编码** | H264 延迟最低，H265 画质更好 | 默认 H264 |
| **码率** | 码率越高画质越好 | 日常 8M，游戏 16M |

### 音频设置

| 选项 | 说明 |
|------|------|
| **音频** | Android 11+ 支持音频转发 |
| **音频源** | 输出（默认）/ 播放 / 麦克风 |
| **音频编码** | AAC / OPUS / FLAC / RAW |
| **音频缓冲** | 值越小延迟越低，值越大越稳定 |

### 录屏设置

| 选项 | 说明 |
|------|------|
| **录制开关** | 开启后录制备份 |
| **录制格式** | MP4（通用）/ MKV / OPUS / FLAC |
| **仅录制不播放** | 后台录制，不显示投屏画面 |
| **时间限制** | 自动停止录制时间 |

### 显示设置

| 选项 | 说明 |
|------|------|
| **画面旋转** | 锁定投屏方向 |
| **保持常亮** | 投屏期间手机不锁屏 |
| **息屏投屏** | 手机黑屏但继续投屏 |
| **虚拟显示** | 在新虚拟屏幕中启动应用 |

### 高级设置

| 选项 | 说明 |
|------|------|
| **键盘模式** | SDK（默认）/ UHID（低延迟）/ Virtual |
| **鼠标模式** | SDK（默认）/ UHID（更精确）|
| **视频源** | 设备屏幕（默认）/ 摄像头 |
| **游戏手柄** | UHID / SDK 模式 |
| **不创建窗口** | 无窗口模式，适合后台录制 |

## 📋 预设配置

应用内置了 5 种预设方案，一键切换：

| 预设 | 适用场景 |
|------|---------|
| ⚖️ **均衡模式** | 日常使用，默认设置 |
| ⚡ **低延迟模式** | 游戏等需要低延迟的场景 |
| 🎨 **高画质** | 录屏演示，最大化画质 |
| 📶 **无线模式** | WiFi 连接时减少带宽 |
| 🎥 **仅录屏** | 后台录制不投屏 |

### 自定义预设

1. 在「设置」页面调整好参数
2. 点侧边栏「预设」
3. 点「保存当前配置」→ 输入名称
4. 下次直接点击预设即可一键应用

## ❓ 常见问题

### Q: 提示 `找不到设备`

1. 确认 USB 调试已开启
2. 换一根数据线（部分线只支持充电）
3. 运行 `adb devices` 检查是否能识别

### Q: 投屏画面卡顿

1. 使用 **USB 有线连接**（最快）
2. 预设选择 **「低延迟模式」**
3. 降低分辨率和帧率

### Q: 没有声音

- 需要 Android 11+ 设备
- 检查设置中**音频**是否开启
- 部分小米机型音频转发可能有兼容问题

### Q: WiFi 连接失败

1. 确认手机和电脑在**同一个网络**
2. 确认手机已开启**无线调试**
3. 尝试用 USB 先连接一次运行 `adb tcpip 5555`

### Q: 关闭按钮点了没反应

- 点 × → 弹出「缩小到托盘」/「关闭应用」对话框
- 点「缩小到托盘」→ 窗口隐藏到系统托盘
- 左键点击托盘图标恢复窗口
- 右键托盘图标 → 退出

### Q: 应用启动报错

- 确保安装了 [Microsoft Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)
- 如果报 `electron-store` 错误，请重新下载最新版本
