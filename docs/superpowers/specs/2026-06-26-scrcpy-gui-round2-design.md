# scrcpy GUI 第二轮改进设计文档

日期: 2026-06-26

## 概述

基于上一轮完善（系统托盘 + 浅色主题 + 窗口记忆），本轮聚焦三个方向：快捷键悬浮窗、投屏延迟优化、设备管理增强。

---

## 一、快捷键悬浮窗

### 需求

在投屏页面（MirrorView）添加一个「⌨️ 快捷键」按钮，点击后弹出一个浮动窗口，展示 scrcpy 所有常用快捷键。点击遮罩层或关闭按钮可关闭。

### UI 设计

- **触发按钮**：在 `ControlBar` 旁边或上方，一个 `n-button` 带键盘图标
- **浮动窗口**：使用 `n-modal` 或 `n-drawer`，宽度约 360px，居中显示
- **内容**：表格形式列出快捷键组合和功能说明，分为几个分组：
  - 窗口控制：全屏 (`Ctrl+f`)、退出全屏 (`Ctrl+x`)
  - 设备控制：息屏 (`Ctrl+o`)、旋转 (`Ctrl+r`)、截屏 (`Ctrl+s`)
  - 剪贴板：电脑→手机粘贴 (`Ctrl+v`)、手机→电脑复制 (`Ctrl+c`)
  - 鼠标：返回（右键）、主页（中键）、最近任务（Ctrl+左键）
  - 音量：音量+ (`Ctrl+↑`)、音量- (`Ctrl+↓`)
  - 其他：显示 FPS (`Ctrl+i`)、打开设置面板 (`Ctrl+Shift+s`)

### 实现

- 创建 `ShortcutHint.vue` 组件
- 在 `MirrorView.vue` 中添加触发按钮和模态框
- 快捷键数据以静态常量定义在组件内（不涉及 IPC）

## 二、投屏延迟优化

### 现状

scrcpy 默认使用 H264 编码，码率 8Mbps，无帧率限制。延迟约 35-70ms（取决于设备编码能力和传输方式）。

### 优化措施

1. **默认编码保持 H264**（已是最低延迟编码）
2. **添加「低延迟模式」预设**：
   - 分辨率限制 1024（`-m1024`）
   - 帧率限制 60fps
   - 视频码率 16Mbps
   - 编码器 H264
3. **USB 连接检测提示**：
   - 启动投屏时，如果是 WiFi 连接，显示提示：「无线连接延迟较高，建议使用 USB」
   - 仅提示一次，不阻断操作
4. **预设配置优化**：更新内置预设，增加「低延迟模式」和「USB 高速模式」

### 非目标

- 无法突破 scrcpy 的 ADB 传输瓶颈
- 不做画面帧率插值或预测渲染
- 不做采集卡/投屏线支持

## 三、设备管理增强

### 需求

- 保存曾连接过的设备信息（序列号、型号）
- 一键重新连接历史设备
- 不自动弹窗（USB 插入不弹提示）

### 实现方案

1. **连接历史持久化**：
   - 使用 `localStorage` 保存最近连接的设备列表（最多 10 个）
   - 每个条目：serial、model、lastConnected 时间戳

2. **UI 展示**：
   - 在首页（HomeView）设备列表下方添加「历史设备」区域
   - 显示已保存的设备型号和序列号
   - 点击直接调用 ADB 连接（如果设备还在的话）

3. **自动记录**：
   - 每次成功连接（投屏启动）时，自动将设备信息存入历史
   - 如果已存在则更新最后连接时间

## 四、验收标准

1. 投屏页面点击「⌨️ 快捷键」→ 弹出快捷键列表悬浮窗
2. 点击遮罩层或关闭 → 悬浮窗消失
3. 预设中有「低延迟模式」→ 应用后分辨率/码率/帧率自动调整
4. 连接过一次设备后 → 首页出现历史设备列表
5. 点击历史设备 → 自动尝试连接
6. USB 插入系统不弹窗
7. 快速设置面板文字显示正常（不挤在一起）
